# AGENTS.md

This file provides guidance to AI coding agents when working with code in this repository.

## Linting

Charts are linted using [chart-testing](https://github.com/helm/chart-testing). CI runs on every PR:

```bash
ct lint --config .ct.yaml
```

`ct lint` only checks charts that have changed relative to `main`. To lint a specific chart directly:

```bash
helm lint charts/<chart-name>
```

To template and inspect rendered output:

```bash
helm template <release-name> charts/<chart-name> -f charts/<chart-name>/values.yaml
```

## Repository Structure

This is a Helm chart mono-repo for Cofide's SPIFFE-based zero-trust infrastructure. There are 7 charts under `charts/`:

| Chart | Purpose |
|---|---|
| `cofide-agent` | Deploys the Cofide Agent (SPIFFE workload identity agent) |
| `cofide-connect` | Deploys the Cofide Connect API server with an Envoy sidecar |
| `cofide-connect-ui` | Deploys the Connect UI frontend with an Envoy sidecar |
| `cofide-observer` | Deploys the Cofide Observer (monitoring/observability) |
| `cofide-trust-zone-operator` | Kubernetes operator for managing `TrustZoneServer` CRs |
| `cofide-tzaas-crds` | Installs the `TrustZoneServer` CRD â€” must be installed before `cofide-trust-zone-operator` |
| `spiffe-enable` | Admission webhook that automates SPIFFE workload identity injection |

## Architecture Notes

**CRD chart pattern**: `cofide-tzaas-crds` places CRDs in `templates/` (not `crds/`) intentionally â€” this enables Helm templating (labels/annotations) and explicit lifecycle management via `helm.sh/resource-policy: keep`. This is the same pattern used by cert-manager and SPIRE helm-charts-hardened.

**Chart dependencies**: No Helm chart dependencies are declared between charts. The only install-order constraint is `cofide-tzaas-crds` before `cofide-trust-zone-operator`.

**SPIFFE throughout**: All charts assume SPIFFE workload identity is available via the CSI driver (`csi.spiffe.io`). The `spiffe-enable` chart requires `cert-manager` as a prerequisite.

**Envoy sidecar pattern**: `cofide-connect` and `cofide-connect-ui` both deploy an Envoy proxy sidecar alongside the main container, configured via a separate `configmap-envoy.yaml` template.

**Trust Zone Operator**: The operator manages TrustZoneServer custom resources (group tzaas.connect.cofide.io/v1alpha1). It includes a Validating Admission Policy, metrics endpoint with certificate, and Gateway API resources (TLSRoute, ReferenceGrant) for Envoy proxy configuration.

## Release Process

Charts are released automatically on push to `main` via `helm/chart-releaser-action`. Each chart is packaged independently and published to GitHub Releases, with a Helm index on `gh-pages`. An aggregated index (merging local charts with Cofide's SPIRE charts) is generated by `.github/actions/aggregate-indexes/merge_indexes.py`.

Both `Chart.yaml` version fields must be bumped when making changes:
- `version`: the chart version (increment on any chart change)
- `appVersion`: the deployed application version (increment when the app version changes)
